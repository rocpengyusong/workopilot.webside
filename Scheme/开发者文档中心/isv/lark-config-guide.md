# LARK 飞书服务配置指引

## 1. 配置概述

本文档详细介绍如何在 Workopilot 平台配置飞书服务，包括飞书开放平台应用创建、Workopilot 服务绑定等完整流程。

### 配置流程概览

```
┌─────────────────────┐    ┌─────────────────────┐    ┌─────────────────────┐
│  飞书开放平台        │ -> │  Workopilot管理后台  │ -> │  接口调用测试        │
│  创建自建应用        │    │  绑定飞书服务        │    │  验证配置正确性      │
└─────────────────────┘    └─────────────────────┘    └─────────────────────┘
```

## 2. 飞书开放平台配置

### 2.1 创建自建应用

1. **登录飞书开放平台**
   - 访问：https://open.feishu.cn/
   - 使用飞书管理员账号登录

2. **创建应用**
   - 进入「管理后台」→「应用管理」
   - 点击「创建自建应用」
   - 填写应用信息：
     - 应用名称：例如「Workopilot消息服务」
     - 应用描述：描述应用用途
     - 应用图标：上传应用图标

3. **获取应用凭证**
   - 进入「凭证与基础信息」页面
   - 记录以下信息：
     - **App ID**：应用唯一标识
     - **App Secret**：应用密钥（点击显示后复制）

> **重要提示**：App Secret 是敏感信息，请妥善保管，不要泄露给他人。

### 2.2 申请接口权限

在飞书开放平台的应用权限管理页面，申请以下必要权限：

| 权限名称 | 权限值 | 使用场景 |
|----------|--------|----------|
| 获取用户基本信息 | contact:user.base:readonly | 通过手机号/邮箱查询用户 |
| 获取用户邮箱 | contact:user.email:readonly | 邮箱ID类型发送消息 |
| 发送消息 | im:message | 发送单聊、群聊消息 |
| 群@消息权限 | im:message:group_at_msg | 群聊消息中@用户 |
| 上传图片/文件 | im:resource | 上传消息资源 |
| 获取群组信息 | im:chat | 向群聊发送消息 |

**申请步骤**：
1. 进入「权限管理」→「接口权限」
2. 搜索上述权限名称
3. 点击「申请权限」
4. 填写申请理由：例如「用于向用户发送业务通知消息」
5. 等待企业管理员审批通过

### 2.3 发布应用

权限申请完成后，需要发布应用：

1. 进入「版本管理与发布」
2. 点击「创建新版本」
3. 填写版本说明
4. 点击「申请发布」
5. 等待企业管理员审批通过

## 3. 可选：事件回调配置

如需接收飞书事件回调（如用户消息回调），需要配置事件订阅。

### 3.1 配置回调地址

1. 进入「事件订阅」页面
2. 点击「添加请求」
3. 填写以下信息：
   - **请求地址**：`https://your-domain.com/openapi/callback/lark/event`
   - **Encrypt Key**：自定义加密密钥（32位字符串）
   - **Verification Token**：自定义验证令牌

### 3.2 记录回调凭证

将以下信息记录下来，后续在 Workopilot 配置时需要使用：
- Encrypt Key
- Verify Token

### 3.3 订阅事件类型

根据业务需求订阅以下事件：

| 事件类型 | 事件说明 | 订阅建议 |
|----------|----------|----------|
| im.message.receive_v1 | 接收消息 | 需要接收用户消息时订阅 |
| application.bot.menu_v1 | 菜单点击 | 使用菜单功能时订阅 |

## 4. Workopilot 服务绑定

### 4.1 进入应用服务管理

1. 登录 Workopilot 管理后台
2. 进入「应用管理」→「应用服务」
3. 找到要绑定的外部应用

### 4.2 添加飞书服务配置

1. 点击「添加服务」
2. 选择服务类型：**LARK（飞书）**
3. 填写服务配置：

| 参数名称 | 参数说明 | 示例值 | 是否必填 |
|----------|----------|--------|----------|
| 服务名称 | 配置标识 | 飞书服务-生产环境 | 是 |
| App ID | 飞书应用ID | cli_xxxxxxxxx | 是 |
| App Secret | 飞书应用密钥 | xxxxxxxxxxx | 是 |
| Encrypt Key | 加密密钥（事件回调） | 32位随机字符串 | 否 |
| Verify Token | 验证令牌（事件回调） | 自定义token | 否 |

4. 点击「确定」保存配置

### 4.3 验证配置

保存后，系统会自动验证配置是否正确：
- 验证 App ID 和 App Secret 是否有效
- 测试获取 tenant_access_token
- 如验证失败，请检查凭证是否正确

## 5. 配置验证

### 5.1 使用接口测试

配置完成后，使用以下步骤验证配置是否正确：

```bash
# 测试发送文本消息
curl -X POST https://api.workopilot.com/openapi/app/lark/message/text \
  -H "x-api-key: sk_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "userId": "+8613800138000",
    "idType": "PHONE",
    "text": "配置测试消息"
  }'
```

### 5.2 预期结果

- 成功响应：`{"code": 0, "message": "success", "data": "om_xxx"}`
- 失败响应：查看错误码，根据错误信息调整配置

### 5.3 常见错误处理

| 错误码 | 错误信息 | 处理方法 |
|--------|----------|----------|
| 30002 | 服务未配置 | 检查是否已完成服务绑定 |
| 99991008 | App ID 无效 | 检查 App ID 是否正确 |
| 99991015 | App Secret 无效 | 检查 App Secret 是否正确 |
| 99991401 | 无接口权限 | 检查飞书开放平台是否已申请权限 |
| 99990320 | 用户不存在 | 检查 userId 是否正确 |

## 6. 生产环境配置建议

### 6.1 环境隔离

建议为不同环境创建独立的飞书应用：

| 环境 | 应用名称 | 建议命名 |
|------|----------|----------|
| 开发环境 | Workopilot开发测试 | Workopilot-Dev |
| 测试环境 | Workopilot测试环境 | Workopilot-Test |
| 生产环境 | Workopilot生产环境 | Workopilot-Prod |

### 6.2 安全建议

1. **凭证管理**
   - 不要将 App Secret 硬编码在代码中
   - 使用环境变量或密钥管理服务存储
   - 定期更换 App Secret

2. **权限最小化**
   - 仅申请业务必需的权限
   - 定期审查已申请的权限
   - 及时撤销不再使用的权限

3. **日志监控**
   - 监控接口调用成功率
   - 记录异常日志便于排查
   - 设置告警规则

### 6.3 多应用支持

如需管理多个飞书应用（不同租户使用不同飞书应用）：
1. 在 Workopilot 中创建多个外部应用
2. 每个外部应用绑定各自的飞书服务配置
3. 使用不同的 API Key 调用

## 8. 常见问题

### Q1: 权限申请需要多久审批通过？

A: 飞书权限审批通常需要1-3个工作日，具体时间取决于企业管理员审批流程。

### Q2: 测试环境可以和生产环境共用一个飞书应用吗？

A: 不建议。建议为不同环境创建独立的飞书应用，避免测试数据影响生产环境。

### Q3: App Secret 泄露了怎么办？

A: 立即在飞书开放平台重置 App Secret，并在 Workopilot 后台更新配置。

### Q4: 如何查看接口调用日志？

A: 在飞书开放平台的应用管理页面，可以查看接口调用日志和错误信息。

### Q5: 发送消息频次有限制吗？

A: 飞书对消息发送频次有限制，具体限制请参考飞书官方文档。建议合理控制发送频率。
