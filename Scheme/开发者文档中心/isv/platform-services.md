# Workopilot 平台服务

## 1. 服务概述

Workopilot 开放平台提供多种平台服务，ISV 可以根据业务需求选择相应的服务进行接入。每个服务对应不同的第三方平台能力，帮助企业快速构建智能化应用。

### 1.1 服务列表

| 服务编码 | 服务名称 | 服务描述 | 适用场景 |
|----------|----------|----------|----------|
| LARK | 飞书服务 | 对接飞书开放平台，提供消息收发、资源管理能力 | 企业协作、消息通知、自动化办公 |
| WE_COM | 企业微信 | 对接企业微信平台，提供消息推送能力 | 企业内部通知、客户服务 |
| DING_TALK | 钉钉 | 对接钉钉开放平台 | 钉钉用户的消息触达 |
| WEMEET | 腾讯会议 | 对接腾讯会议平台 | 会议管理、会议提醒 |

## 2. LARK 飞书服务

### 2.1 服务简介

飞书是字节跳动推出的一站式企业协作平台，Workopilot 提供完整的飞书服务对接能力，包括：

- **文本消息**：发送纯文本消息
- **图片消息**：发送图片内容
- **富文本消息**：发送带格式的富文本内容
- **卡片消息**：发送交互式卡片，支持按钮、表单等交互元素
- **资源管理**：图片、文件的上传和下载

### 2.2 核心功能

| 功能模块 | 接口路径 | 说明 |
|----------|----------|------|
| 文本消息 | POST /openapi/app/lark/message/text | 发送文本消息 |
| 图片消息 | POST /openapi/app/lark/message/image | 发送图片消息 |
| 富文本消息 | POST /openapi/app/lark/message/post | 发送富文本消息 |
| 卡片消息 | POST /openapi/app/lark/message/card | 发送卡片消息 |
| 卡片消息(模板) | POST /openapi/app/lark/message/cardById | 使用卡片模板发送消息 |
| 上传图片 | POST /openapi/app/lark/resource/image/upload | 上传图片获取 image_key |
| 上传文件 | POST /openapi/app/lark/resource/file/upload | 上传文件获取 file_key |
| 下载图片 | GET /openapi/app/lark/resource/image/download | 通过 image_key 下载图片 |
| 下载文件 | GET /openapi/app/lark/resource/file/download | 通过 file_key 下载文件 |

### 2.3 应用场景

- **订单通知**：订单状态变更时向用户发送飞书消息通知
- **审批提醒**：审批流程中的节点提醒和结果通知
- **营销推广**：向目标用户发送营销消息和活动信息
- **客服支持**：自动回复用户咨询，提供智能客服能力
- **数据推送**：定期向用户推送报表、统计数据

### 2.4 用户标识类型

飞书服务支持多种用户标识类型，系统会自动处理 ID 转换：

| 标识类型 | 说明 | 示例值 |
|----------|------|--------|
| open_id | 飞书用户 open_id | ou_xxx |
| user_id | 飞书用户 user_id | xxx |
| chat_id | 飞书群聊 ID | oc_xxx |
| union_id | 飞书用户 union_id | on_xxx |
| phone | 手机号（自动转换） | +8613800138000 |
| email | 邮箱（自动转换） | user@example.com |

## 6. 服务接入流程

### 6.1 标准接入流程

```
┌─────────────────────────────────────────────────────────────┐
│                    1. 注册外部应用                            │
│                   在 Workopilot 管理后台                     │
│                   创建外部应用，获取 API Key                  │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                    2. 配置平台服务                            │
│              在对应平台创建应用并获取凭证                      │
│         (如飞书开放平台创建应用获取 App ID/Secret)            │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                    3. 绑定服务配置                            │
│           在 Workopilot 应用服务中添加平台配置                │
│          填写 App ID、App Secret 等必要信息                  │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                    4. 调用接口测试                            │
│              使用 API Key 调用对应服务接口                    │
│                   验证配置是否正确                            │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                    5. 上线部署                                │
│              配置生产环境凭证，正式发布应用                    │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 注意事项

1. **凭证安全**：App Secret 等敏感信息请妥善保管，定期更换
2. **回调地址**：如需接收平台回调（如飞书事件回调），需配置正确的回调地址
3. **权限申请**：在平台创建应用时，请根据实际业务申请必要的接口权限
4. **测试验证**：正式上线前，建议在测试环境完成完整的功能验证

## 7. 服务配置参数说明

### 7.1 LARK 飞书服务配置

| 参数名称 | 参数说明 | 是否必填 | 获取方式 |
|----------|----------|----------|----------|
| App ID | 飞书应用 ID | 是 | 飞书开放平台应用详情页 |
| App Secret | 飞书应用密钥 | 是 | 飞书开放平台应用详情页 |
| Encrypt Key | 加密 Key | 否 | 飞书开放平台事件回调配置 |
| Verify Token | 验证 Token | 否 | 飞书开放平台事件回调配置 |

### 7.2 权限申请清单

使用飞书服务时，请在飞书开放平台申请以下权限：

| 权限名称 | 权限描述 | 使用场景 |
|----------|----------|----------|
| im:message | 发送消息权限 | 发送各类消息 |
| im:message:group_at_msg | 群@消息权限 | 群聊消息中@用户 |
| im:resource | 上传资源权限 | 图片/文件上传 |
| im:chat | 获取群组信息 | 向群聊发送消息 |
| contact:user.base:readonly | 获取用户基本信息 | 通过手机号/邮箱查询用户 |
| contact:user.email:readonly | 获取用户邮箱 | 邮箱ID类型发送消息 |

## 8. 常见问题

### Q1: 如何选择合适的平台服务？

A: 根据您的目标用户群体选择：
- 如果目标用户主要使用飞书进行协作，选择 LARK 服务
- 如果是企业内部应用，可能需要 WE_COM 企业微信服务
- 可以同时接入多个服务，覆盖不同用户群体

### Q2: 一个应用可以同时绑定多个服务吗？

A: 可以。一个外部应用可以绑定多个平台服务，根据业务需求灵活选择。

### Q3: 服务绑定后可以更换吗？

A: 可以。在应用服务管理中可以随时修改或删除服务配置，修改后立即生效。

### Q4: 服务调用失败如何处理？

A: 请检查：
1. 服务是否已正确配置并绑定到应用
2. 平台凭证（App ID/Secret）是否有效
3. 是否申请了必要的接口权限
4. 查看接口返回的错误码和错误信息
