# 核心功能详解

## 1. 伴随式审核工作台 (Browser Plugin)

审核员的日常工作不再面对我们的系统后台，而是继续使用 SAP、用友或 OA。

- **自动激活**：插件自动识别当前 URL（如差旅报销页），智能唤醒。
- **红绿灯机制**：
    - 🟢 **PASS**：风险检测通过，建议放行。
    - 🔴 **BLOCK**：发现阻断性风险（如发票抬头错误），禁止提交。
    - 🟡 **WARN**：发现警示性风险（如住宿费略微超标），需人工复核。
- **证据溯源**：点击风险项，直接弹窗展示 **“发票切片 vs 填报数据”** 的对比图，AI 的判断依据一目了然。

## 2. 智审策略编排中心 (Strategy Studio)

系统的“大脑”构建工厂，专为策略管理员设计。

### 规则配置 IDE
一个类似 VSCode 的低代码配置环境：
- **Prompt 工程**：直接编写自然语言指令，支持变量插入（如 `{{invoice_amt}}`）。
- **动态数据源**：支持勾选 **附件分类白名单**（如仅提取“酒店水单”），精准控制 AI 注意力。
- **工具挂载**：为规则挂载 **MCP 工具**（如汇率查询、企业工商查询）。

### 全链路仿真沙箱
在规则发布前，进行 100% 真实的模拟测试：
- **模拟输入**：上传真实的附件图片，编辑模拟的单据 JSON。
- **透视运行**：查看 AI 的思考链（CoT）、Token 消耗量及中间变量快照。

## 3. 混合制度引擎 (Hybrid Policy Engine)

将企业的“死制度”变成“活数据”。

- **结构化参数**：对于刚性指标（如：北京，M3级，限额800元），AI 自动提取为结构化数据，进行精确数学比对。
- **RAG 语义检索**：对于柔性原则（如：是否符合业务招待必要性），利用向量检索技术，从 PDF 制度文档中查找原文依据。

## 4. 能力与集成中心 (Capabilities Hub)

- **附件分类中心**：支持 OCR+LLM 或纯视觉模型（Vision Model）两种路线，支持多页合同的结构化拆分（封面/正文/签字页）。
- **工具生态**：
    - **Native Tools**：系统内置的高性能 C# 方法。
    - **MCP Services**：兼容 Model Context Protocol 协议，轻松对接外部 Python/Node.js 服务。
